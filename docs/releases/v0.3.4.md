# Gonimbus v0.3.4 Release Notes

**Release Date**: 2026-06-27
**Status**: Google Cloud Storage Provider and Library-Exposure Foundation

Gonimbus v0.3.4 widens the provider matrix beyond S3 and `file://` for the first
time: Google Cloud Storage (`gs://`) becomes a first-class crawl/inspect source
and `transfer reflow` destination. GCS rides the existing provider-dispatch seam
and reuses the same adaptive-concurrency and IfAbsent-capability model already
documented for S3, so existing workflows extend to GCS rather than forking. The
release also lands the quieter structural groundwork for exposing the reflow
engine as an embeddable library, with CLI behavior unchanged.

## Highlights

- **Google Cloud Storage provider** — `gs://` sources and reflow destinations
  across inspect/index/tree/stream/content/doctor and `transfer reflow`, with the
  same IfAbsent honored/probe-status summary fields as S3 and the same adaptive
  concurrency model.
- **GCS authentication** — Application Default Credentials and service-account
  keys under the established credential-source discipline (no URI- or
  manifest-sourced credential filepaths); `STORAGE_EMULATOR_HOST` is test-only.
- **Library-exposure foundation (Experimental)** — `pkg/reflow` exposes typed
  records, the adaptive-concurrency substrate, and redaction helpers; the CLI
  still drives the internal path. The refactor completes in a later release.
- **Reflow probe operations bounded by adaptive concurrency** — probes now back
  off with copies under provider throttling.
- **Cross-version checkpoint resume fix** — `--resume-run` works again across the
  v0.3.2 → v0.3.3 upgrade boundary.

## Google Cloud Storage is here

The support bar: read (List/Head/Get/range) and reflow destination
(Put + conditional IfAbsent). GCS reports IfAbsent honored/probe-status the same
way S3 does and maps throttling (`RESOURCE_EXHAUSTED`/429/503) to
`provider.ErrThrottled`, so the adaptive `--parallel` behavior and the
capability-aware collision fallback both apply unchanged.

One deliberate limitation: GCS uses generation-based preconditions rather than
ETag `If-Match`, so ETag-based `If-Match` conditional writes are unsupported and
`--on-collision overwrite-if-source-newer` is unavailable on GCS destinations
(it fails closed). IfAbsent-based `skip-if-duplicate` is fully supported.

## The quieter story: library foundation

`pkg/reflow` now exposes the reflow engine's lower-level building blocks — typed
JSONL records, the adaptive-concurrency substrate, and provider-error redaction
helpers — as an **Experimental** surface per
[`docs/api-stability.md`](../api-stability.md). The `transfer reflow` CLI is
unchanged and continues to drive the internal path; the data/decision-plane
migration onto `pkg/reflow` completes in a later release. This is aimed at
library consumers; CLI users see no behavior change.

## Fixes and housekeeping

- **Checkpoint resume across versions:** the op-checkpoint `no_adaptive` config
  field is now omitted when unset, restoring `--resume-run` for resumable
  checkpoints written by pre-v0.3.3 builds (previously failed closed with an
  identity mismatch), guarded by a cross-version fingerprint compatibility test.
- **Reflow probe bounding:** standalone provider probes (source/destination HEADs
  and body-compare reads) now acquire adaptive-concurrency slots like copies, so
  backoff under throttling reduces probe pressure too; acquisitions are sequential
  and never nested (deadlock-free at the concurrency floor).
- **Dependencies:** `golang.org/x/net` upgraded to v0.56.0.
- **Docs:** corrected broken `transfer reflow` rewrite examples in the shipped
  v0.2.1 / v0.2.3 release notes.

## Upgrade notes

- No CLI behavior change. Existing S3 and `file://` workflows are unaffected.
- GCS credentials resolve via ADC / service-account keys — no new manifest
  credential fields, consistent with the credential-source discipline.
- The new `pkg/reflow` surface is **Experimental**; embedders should expect it to
  evolve until the migration completes.
