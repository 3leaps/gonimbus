# Gonimbus v0.3.3 Release Notes

**Release Date**: TBD
**Status**: Draft release package

Gonimbus v0.3.3 is the release train for adaptive transfer reflow concurrency,
priority probe fallback handling, reflow error-surface hardening, and the final
release-package documentation for those changes.

This page is intentionally scaffolded for the release-doc handoff. Technical
drafts should land here first, then the shorter `RELEASE_NOTES.md` entry can be
collapsed from the final wording.

## Highlights

- Adaptive `transfer reflow --parallel` semantics: requested ceiling,
  resource-capped effective ceiling, default-on AIMD behavior, and
  `--no-adaptive` fixed at the effective ceiling.
- Priority XML XPath probe fallback: ordered `xpath_priority` resolution and
  fail-closed quarantine handling for truncated fallback values.
- Reflow error hardening: classified resumable abort causes plus redaction for
  published provider-error surfaces.
- S3 auth and transport polish: hermetic signing coverage for profile/default
  credential paths and opt-in S3 transport tuning from transfer reflow.

## Library API

This release has additive Stable API changes and no Stable breaks:

- `pkg/provider/s3.Config` adds optional `MaxIdleConnsPerHost` and
  `MaxConnsPerHost` transport sizing fields. Zero values preserve AWS SDK
  defaults.
