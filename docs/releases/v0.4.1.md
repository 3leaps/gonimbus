# Gonimbus v0.4.1 Release Notes

**Release Date**: 2026-07-18
**Status**: SQLite Independence + Streaming Durable Publication

Gonimbus v0.4.0 made the **durable index format the default** build artifact.
v0.4.1 completes the operator story around that default in two directions:

1. **SQLite independence.** A durable-only default build no longer needs an
   `index.db` for everyday local work. `query`, `list`, `stats`, `doctor`, and
   `enrich-with-head` all read the durable substrate natively, and a
   machine-stable build receipt lets automation hand off the exact committed
   snapshot without list heuristics.
2. **Memory-bounded streaming publication.** `index build` now publishes durable
   snapshots through a streaming compaction path that never materializes the
   full row set in memory, under finite, operator-tunable capacity budgets.

SQLite remains a first-class **compatibility path**. Existing `index.db` files
are not rewritten or invalidated. Use `--format sqlite` when a workflow needs
the canonical `index.db` (SQLite-only surfaces such as `query --since-run`,
`stats --prefixes`, and full `--resume-run` recovery), or `--format both` for a
durable publication plus a per-run SQLite parity-verification projection.

Operator walkthrough: [Durable Index Format](../user-guide/durable-index.md).

## Highlights

- **Durable-native `index query`** — durable segments are scanned natively
  through a format-aware reader seam (`pkg/indexreader`), with no ephemeral
  SQLite bridge. Plain query streams verified JSONL as rows arrive, with
  same-open / same-bytes digest verification. `query --since-run` fails closed on
  `durable-v2`.
- **Build receipt and pinned-run handoff** — `index build --json` emits a
  terminal `gonimbus.index.build_result.v1` receipt (status, requested and
  committed formats, full set/run/scope identity, counts, durable
  `manifest_sha256`). A pinned durable query (`--index-set` + `--run-id`) opens
  the run complete marker directly and never follows `latest.json`, so a later
  `latest` advance cannot silently switch the snapshot under a consumer.
- **Format-aware inventory and diagnostics** — `index list`, `index stats`, and
  `index doctor` work for durable-only sets with no `index.db`. `doctor`
  inspects the SQLite and durable substrates independently, so a broken durable
  marker stays discoverable and reports unhealthy rather than vanishing.
- **Durable `index enrich-with-head`** — HEAD enrichment for durable-only sets
  runs under an OS-level exclusive write lease. Prior segments stay immutable,
  and unobserved keys are never tombstoned.
- **Managed background builds** — `index build --background` runs a managed
  durable build via a canonical effective invocation and exact child-process
  replay, with deduplication, leases, recovery, terminal receipts, and job
  log/API surfaces. Credential-bearing, signed, and metadata invocation material
  is rejected before persistence.
- **Whole-set garbage collection** — `index gc --dry-run` (`--max-age`,
  `--keep-last`, optional `--json`) produces a deterministic, format-aware
  reclamation plan that groups SQLite and durable artifacts by full index-set
  identity and retains conservatively when safety cannot be proven. Leased
  execution and a canonical SQLite authority binding land alongside the planner.
- **`index migrate-match-scope`** — a fail-closed migration of prefix-shaped
  `build.match.includes` into explicit `build.scope` `prefix_list`, verified by
  LIST-plan equality and an independent LIST-reachable projection proof.
- **Streaming durable publication** — the default `index build` path compacts
  sealed journals against the verified parent without holding the full row set
  in memory, under operator-tunable capacity budgets.
- **`transfer reflow` stdin `--parallel` restored** — live stdin reflow copies
  route through the CLI worker pool again (a v0.3.5–v0.4.0 regression).

## What operators should do differently

Most v0.4.0 workflows are unchanged. The new local surfaces mean a durable-only
default build is now self-sufficient for everyday inspection:

```bash
# Default durable build (no index.db) — now queryable, listable, and
# diagnosable natively
gonimbus index build --job index.yaml

# Machine-stable receipt for automation handoff
gonimbus index build --job index.yaml --json

# Query the durable snapshot directly (streaming verified JSONL)
gonimbus index query --index-set idx_... --prefix data/

# Pin the exact committed run (never follows latest.json)
gonimbus index query --index-set idx_... --run-id run_...

# Inventory and diagnostics on durable-only sets
gonimbus index list
gonimbus index stats --index-set idx_...
gonimbus index doctor --index-set idx_...

# HEAD enrichment for a durable-only set (exclusive write lease)
gonimbus index enrich-with-head --index-set idx_...

# Managed background build with recovery + terminal receipt
gonimbus index build --job index.yaml --background

# Dry-run whole-set reclamation plan (no executor side effects)
gonimbus index gc --dry-run --keep-last 3 --json

# Migrate a prefix-shaped match.includes job to explicit build.scope
gonimbus index migrate-match-scope --job index.yaml
```

### Streaming publication and capacity budgets

`index build` on the durable path now publishes through a streaming compaction
engine: sealed crawl journals are compacted against the verified parent snapshot
through a spill/merge current-state source and a streaming segment-set writer,
so the full row set is never materialized in memory. **The emitted rows, segment
artifacts, manifests, and digests are identical to the prior materialized
path** — the only difference is bounded memory during publication.

Three operator knobs bound scratch and journal-line size. Precedence for each:
**CLI flag > environment > app config > built-in default.**

| Knob                      | Flag                    | Env                            | Config                      | Default             |
| ------------------------- | ----------------------- | ------------------------------ | --------------------------- | ------------------- |
| Merge scratch ceiling     | `--spill-workspace-max` | `GONIMBUS_SPILL_WORKSPACE_MAX` | `index.spill.workspace_max` | **16 GiB**          |
| Max single journal-record | `--spill-record-max`    | `GONIMBUS_SPILL_RECORD_MAX`    | `index.spill.record_max`    | **16 MiB**          |
| Scratch directory         | `--spill-root`          | `GONIMBUS_SPILL_ROOT`          | `index.spill.root`          | beside run journals |

- Size the workspace ceiling to the **corpus** (a successive build stages the
  full prior index state before merging). Point `--spill-root` at a disk with
  room for that ceiling.
- The single-record bound is payload-exact (line terminators are framing) and is
  enforced identically across sealed-journal validation, the streaming scan, and
  spill-run attestation. Very wide/dense crawl-prefix plans may need a higher
  record ceiling.
- Invalid or unrepresentable budgets refuse with typed errors **before** any
  crawl, journal read, or side effect.
- Budget exhaustion refuses fail-closed with **no authority advance** — a
  refused build never clobbers `latest`.
- CLI spill flags are **not** forwarded to `--background` jobs; use env or
  config so the managed child inherits them.
- Segment packing remains a fixed **500,000 rows/segment** engine lever (not an
  operator setting).

Streaming segment progress and workspace peak/ceiling diagnostics render on
stderr (counts and sources only). Progress is best-effort and can never fail a
build. During streaming publication, segment progress reports against a
streaming total of `0` by contract (the full count is not known until the tail).

Full operator guide (examples, config YAML, refusal rules):
[Durable Index Format — streaming capacity budgets](../user-guide/durable-index.md#streaming-capacity-budgets-operator-tunable).

### Compatibility notes

- No format or schema break. Existing durable-v2 and `index.db` artifacts remain
  readable; the streaming path publishes the same durable-v2 contract.
- `query --since-run`, `stats --prefixes`, and full `--resume-run` recovery
  remain SQLite-only — use `--format sqlite` or `--format both`.
- `index gc` executes against sets with a canonical SQLite authority binding;
  durable artifacts are included in whole-set planning.
- `--format both` publishes durable as canonical and builds a run-scoped SQLite
  projection purely as per-run parity verification (not a canonical `index.db`).

## Internal-render framing (important)

Durable-v2 remains a full-fidelity **internal render** for trusted operator and
pipeline use. It is **not** a reduced-trust, de-identified, or third-party
publication format. A durable hub export is not a disclosure-controlled share
format; rendering a durable index for any reduced-trust or third-party context
is out of scope for this format.

## Integrity, not authenticity

The content digests carried by durable manifests and segments (and by sealed
journals) are **content-integrity / tamper-evidence** checksums. They detect
corruption or modification of the bytes they cover. They are **not** statements
of author authenticity or provenance, and no recovery, journal, or durability
claim in this release should be read as certifying who produced a record.

## Upgrade notes

```bash
go install github.com/3leaps/gonimbus/cmd/gonimbus@v0.4.1
```

1. Install v0.4.1. Existing durable and SQLite artifacts are read as-is; no
   migration is required.
2. If you previously kept `--format both` (or SQLite) only because durable sets
   were not locally queryable, you can now run, query, list, and diagnose
   durable-only default builds directly.
3. For large builds, the streaming path bounds publication memory automatically.
   Tune the workspace/record budgets only if a build refuses on the default
   ceilings; keep `--format sqlite` for the SQLite-only surfaces above.
4. Read [Durable Index Format](../user-guide/durable-index.md) for the full
   operator map (durable-native surfaces, budgets, parity semantics, SQLite
   escape hatches).

## Library consumers

- `pkg/indexreader` (**Experimental**) is the format-aware reader seam behind
  durable-native query, list, stats, and doctor. Callers resolve a reader from
  markers and stream verified rows.
- `pkg/indexbuild` remains the durable build engine surface (**Experimental**);
  the streaming publication path is internal to the engine and does not change
  its injected-provider contract.
- `pkg/scope` (**Experimental**) exposes `PlanMatchPrefixMigration` behind
  `index migrate-match-scope`; it is storage-free (ADR-0006) with identity
  evidence injected from the CLI adapter.
- `pkg/reflow` remains **Experimental** (no Stable promotion and no Stable
  library API break in this cut).
- Hub consumers should continue to branch on `sqlite-v1` / `durable-v2` markers
  and reject unknown formats.
