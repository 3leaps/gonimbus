# Gonimbus v0.2.0 Release Notes

**Release Date**: 2026-05-20
**Status**: Library-Enabling and Scaling — Far More Complex Reflow Patterns

## Overview

Gonimbus v0.2.0 grows the tool along three axes simultaneously while keeping its core promise intact: predictable, prefix-first crawls over object storage at scale, with JSONL-first outputs.

The three chapters of this release:

1. **Library enablement** — a stable public surface for Go consumers, starting with the URI parser, a documented library-consumer contract, and reliable version stamping for `go install`-built binaries.
2. **Reflow + probe sophistication** — derived variables, mixed-segment rewrites, lookup-table transforms, mirrored provenance sidecars, the Hive-partition docs pattern, and content-fingerprint deduplication. The reflow engine answers a much wider class of "reorganize this content as I describe it" questions in a single pipeline.
3. **Scaling correctness** — atlas phase A (derived views across completed indexes), conditional `latest.json` CAS, atomic conditional puts, and parallel-race arbitration on shared destination keys. Behavior stays right as the substrate, the parallelism, and the dataset all grow.

For the comprehensive list of changes, see [CHANGELOG.md](../../CHANGELOG.md#020---2026-05-20).

## Chapter 1: Library enablement

### The shift

Through v0.1.x, gonimbus was a CLI-first tool with a private library shape — embedded use was possible but consumers had to reach into `internal/` and copy patterns. v0.2.0 begins promoting a stable public surface so Go modules can depend on gonimbus directly. The first downstream consumer (sumpter, in fulmenhq) is the forcing function.

### What's new

#### Public URI parser (`pkg/uri`)

```go
import "github.com/3leaps/gonimbus/pkg/uri"

ou, err := uri.ParseObjectURI("s3://my-bucket/some/prefix/object.json")
if err != nil {
    return err
}
fmt.Println(ou.Scheme, ou.Bucket, ou.Key)
```

The same parser the CLI has been using for v0.1.x is now a public package with a stable signature. Consumers no longer copy-paste from `internal/` or import command code to handle the parsing.

#### Library-consumer config contract docs

[`docs/library-consumers.md`](../library-consumers.md) documents the patterns gonimbus expects of embedding apps:

- Credential resolution and env-var precedence
- Hermetic-embedder posture (no env-var eagerness; no credential logging)
- Dep-tree boundary (what gets pulled into your `go.mod`)
- Provider construction (`s3.NewProvider` and friends)
- Multi-provider isolation when one app talks to multiple S3-compatible substrates

#### `go install`-built binaries report the correct version

```bash
$ go install github.com/3leaps/gonimbus/cmd/gonimbus@v0.2.0
$ gonimbus version
gonimbus 0.2.0
```

Previously, `go install`-built binaries reported `dev` because the version string was sourced only from `-ldflags -X main.version=...` injection. v0.2.0 introduces `internal/buildinfo` with a three-tier resolution chain: ldflags overrides → `runtime/debug.ReadBuildInfo` (covers `go install module@vX.Y.Z`) → embedded `VERSION` file (covers `go install ./cmd/...` from a working tree). A regression-guard integration test builds the binary with no ldflags and asserts the reported version matches `VERSION`.

## Chapter 2: Reflow + probe sophistication

### The shift

The v0.1.7 reflow engine answered "reorganize this content based on a probed field." v0.2.0's engine answers "reorganize this content based on several probed fields, with derivations, lookups, mixed-segment rewrites, mirrored sidecars, and tie-broken canonical selection — all in one pipeline." The engine is still prefix-first and still JSONL-first; it just composes more.

### What's new

#### Derived variables (`derived:`)

`content probe` now computes declared `derived` variables from extracted values via composable transforms:

```yaml
extract:
  - name: date
    xpath: //Record/Date

derived:
  - name: year
    from: date
    transform: substring
    start: 0
    length: 4
  - name: month
    from: date
    transform: substring
    start: 5
    length: 2
```

Available transforms: `substring`, `regex_capture`, `format`, `pad`, `lowercase`, `uppercase`.

#### Mixed rewrite segments

`transfer reflow` renders one placeholder with literal prefix/suffix inside a single path segment:

```yaml
rewrite:
  destination_template: 'reflowed/store-{store_id}/year={year}/month={month}/{rel_key}'
```

In v0.1.x each placeholder needed its own segment (`{year}/{month}/`); v0.2.0 supports `year={year}/month={month}/`, which composes cleanly with Hive-style partition layouts.

#### Lookup transform

For table-driven derivations (e.g. "map this store ID to a region code"):

```yaml
derived:
  - name: region
    from: store_id
    transform: lookup
    match: prefix
    table:
      '700': north
      '720': west
      '730': south
```

Match modes: `regex`, `prefix`, `exact`. `content probe --rewrite-from <var>` lets probe recipes derive values directly from source-key captures (e.g. for `store_id` parsed from the key path) before `transfer reflow` renders destination keys.

#### Provenance sidecars + mirrored placement

Every reflowed object now writes a `.provenance.jsonl` sidecar that records source URI, derived fields, and the rewrite path used. Default placement is sibling-to-destination; `--provenance-sidecar-root` allows sidecars to be written under a separate same-bucket root while preserving the provider-relative `provenance.key` contract — useful when the destination tree is consumed by tooling that does not expect sidecar noise.

#### Collision modes refined

```bash
gonimbus transfer reflow \
  --on-collision quarantine \
  --collision-quarantine-prefix 'quarantine/' \
  ...
```

`--on-collision skip-if-duplicate` is the clearer default (replaces the older `--on-collision log` alias, now deprecated). `--on-collision quarantine` writes collisions to an explicit quarantine prefix instead of dropping them. Collision records emit a nested `collision` object replacing the legacy flat fields — see Upgrade Notes below.

#### Hive-style partition emission pattern

[`docs/user-guide/examples/`](../user-guide/examples/) documents the recommended pattern for emitting Hive-style partition layouts (`category={x}/date={y}/`) by composing `extract` + `derived` + mixed-segment rewrites. No per-pattern engine changes were needed; the new building blocks compose into the pattern naturally.

#### Canonical-by-ETag query mode

```bash
gonimbus index query 's3://bucket/prefix/' --canonical-by-etag
```

Emits one canonical object per non-empty ETag group with deterministic tie-breaks, mixed empty-ETag passthrough records, output-record-count `--count` / `--limit` semantics, and optional `--include-alternates` audit detail. Solves content-fingerprint deduplication at query time when the same content is re-uploaded under different keys.

## Chapter 3: Scaling correctness

### The shift

As datasets pass 10M+ objects and pipelines run multiple workers concurrently, the questions stop being "does this feature work?" and start being "does this feature stay right when the substrate, the parallelism, and the dataset all push back?" v0.2.0 lands the primitives that make those answers `yes`.

### What's new

#### Local atlas phase A

```bash
gonimbus atlas build --index-set idx_<sha256>
```

Produces a derived view across a completed index for cross-run analytics (object counts by prefix, byte distributions, etc.) without re-scanning the substrate. Phase A scope: local-only build, single-index input. Cross-index aggregation and remote-hub-driven atlas are tracked for later releases.

#### Conditional `latest.json` CAS

`gonimbus index export` now uses `If-Match` / `If-None-Match` on `latest.json` writes against substrates that support it, giving fail-closed publish semantics. Best-effort last-writer-wins fallback is preserved for v0.1.x-compatible hubs.

#### Atomic conditional puts

Provider-level `If-None-Match: *` for objects requiring race-safe creation. Opt-in via provider capability detection — operators get the safety on substrates that enforce it, and don't pay a silent-no-op penalty on substrates that don't.

#### Parallel-race arbitration

When `transfer reflow --parallel N>1` routes multiple source objects to the same destination key, a single gonimbus process arbitrates per destination key before issuing conditional writes. The first object lands; byte-identical race losers emit `status=skipped reason=collision.duplicate` instead of redundant `complete collision=no` records. Protects the single-run operator path even on S3-compatible substrates that accept but do not enforce `If-None-Match: *`. Cross-process and cross-operator races still depend on substrate-level conditional-write support.

#### Opt-in build summary

```bash
gonimbus index build --job manifest.yaml --summary
```

Emits a per-run JSONL summary (object counts, byte totals, prefix breakdown) without changing default streaming behavior. Useful for operator-side aggregation and for pipeline-control logic that needs end-of-run signal without parsing the full output stream.

#### Until-resolved probe reads

`content probe` reads until a configurable resolution condition is met rather than fixed byte ranges, reducing wasted reads on variable-length headers and improving throughput on heterogeneous content.

#### `doctor` endpoint + region flags

```bash
gonimbus doctor --provider s3 --endpoint https://s3.example.com --region us-east-2
```

Explicit probe-target overrides for cross-account / cross-endpoint validation when the default credential chain would resolve to the wrong place.

## Upgrade notes

### Breaking changes

**Reflow collision flat fields removed.** The legacy fields `collision_kind`, `collision_etag`, and `collision_size_bytes` are no longer emitted in `gonimbus.reflow.v1` records. Audit tooling must read the nested `collision` object as the sole representation. The Phase A warning window has expired:

```jsonl
{ "schema": "gonimbus.reflow.v1", ..., "collision": { "kind": "duplicate", "etag": "abc123", "size_bytes": 12345 } }
```

If your tooling reads the flat fields, update field accessors to walk into `.collision.kind` / `.collision.etag` / `.collision.size_bytes`. Schema docs: [`schemas/gonimbus/v1.0.0/`](../../schemas/gonimbus/v1.0.0/).

### Deprecations

**`--on-collision log` is deprecated.** Use `--on-collision skip-if-duplicate` (same behavior, clearer name). The alias is retained for one minor release and will be removed in v0.3.0.

### Operator action

- Re-run `go install github.com/3leaps/gonimbus/cmd/gonimbus@v0.2.0` to pick up the version-stamping fix; older binaries built via `go install` report incorrect version strings.
- If you maintain a downstream CI workflow that pins the gonimbus runner image, update to `goneat-tools-runner-glibc:v0.4.2` (see [`docs/development/ci.md`](../development/ci.md) for the GOPATH and bash-shell workarounds the v0.3+ runner family requires).
