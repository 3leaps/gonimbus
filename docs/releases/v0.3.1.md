# Gonimbus v0.3.1 Release Notes

**Release Date**: 2026-06-14
**Status**: Embedded S3 Auth Controls, Toolchain Pinning, and Dependency Refresh

Gonimbus v0.3.1 is a focused release for Go embedders and release operators.
It adds explicit S3 construction modes for unsigned public reads and
caller-managed AWS credential providers, pins the module toolchain directive to
the release-lane Go patch version, refreshes the past-cooling dependency set,
and removes implicit forwarded-IP trust from the default server router.

## Embedded S3 Auth Controls

Embedded consumers can now construct S3 providers without relying on ambient
process credentials. `pkg/provider/s3.Config` adds two explicit modes:

- `Anonymous: true` for unsigned public-bucket read requests.
- `CredentialsProvider` for caller-managed AWS SDK credential handles.

Credential precedence is now explicit:

1. Anonymous unsigned reads.
2. Injected AWS SDK `CredentialsProvider`.
3. Static `AccessKeyID` plus `SecretAccessKey`.
4. AWS shared config profile.
5. AWS SDK default credential chain.

Anonymous mode is mutually exclusive with every credential source. It does not
fall back to environment variables, profiles, web identity, instance metadata,
or any other ambient AWS credential source. Read operations send no
`Authorization` header.

```go
cfg := s3.Config{
	Bucket:    "public-bucket",
	Region:    "us-east-1",
	Anonymous: true,
}
provider, err := s3.New(ctx, cfg)
```

Applications that already manage AWS credential handles can inject them:

```go
cfg := s3.Config{
	Bucket:              "managed-bucket",
	Region:              "us-east-1",
	CredentialsProvider: yourProvider,
}
provider, err := s3.New(ctx, cfg)
```

When `CredentialsProvider` is set, gonimbus skips lower-priority credential
sources. If profile-derived region, endpoint, or path-style settings are needed
alongside an injected provider, pass those values directly in `s3.Config`.

## Anonymous Mode Fails Closed for Writes

Anonymous S3 construction means "unsigned public read", not public write.
S3 write methods such as `PutObject`, `DeleteObject`, conditional PUT, and
multipart upload fail before issuing provider requests. The returned error
joins `provider.ErrAnonymousReadOnly` with `provider.ErrAccessDenied`, and
`provider.IsAnonymousReadOnly` can be used for stable classification.

This keeps embedded applications from accidentally converting read-only public
bucket inspection into a write-capable provider just because ambient
credentials are available in the host process.

See [docs/library-consumers.md](../library-consumers.md#credential-injection)
for the supported embedded-use contract and examples.

## Dependency and Toolchain Refresh

v0.3.1 pins the module toolchain directive:

```go
toolchain go1.26.4
```

That matches the CI, dependency-security, and release workflow Go patch version
used for release evidence.

The dependency refresh is bounded to modules that cleared the release cooling
policy:

this release. v0.3.1 keeps `golang.org/x/net v0.55.0`; dependency-security
evidence remains clean at the high-and-critical gate.

## Server Request IP Posture

The default server router no longer installs `chi/middleware.RealIP`.
That middleware rewrites `RemoteAddr` from caller-supplied `X-Forwarded-For`
and `X-Real-IP` headers. Gonimbus does not currently use rewritten client IPs
for access control, rate limiting, or audit decisions, so the default posture is
now to leave `RemoteAddr` as the network peer address.

If a deployment needs proxy-aware client IPs later, that should be added as
explicit trusted-proxy configuration rather than implicit forwarding-header
trust.

## Library API

This release has additive Stable API changes and no Stable breaks:

- `pkg/provider/s3.Config` adds `Anonymous`.
- `pkg/provider/s3.Config` adds `CredentialsProvider`.
- `pkg/provider` adds `ErrAnonymousReadOnly` and `IsAnonymousReadOnly`.

Existing static-key, profile, and SDK-default-chain construction continues to
work with the documented precedence.

## Upgrade

```bash
go install github.com/3leaps/gonimbus/cmd/gonimbus@v0.3.1
```

CLI workflows remain compatible with v0.3.0. Go embedders using
`pkg/provider/s3` can opt into anonymous public reads or injected SDK
credentials where those modes better match the host application's credential
boundary.
