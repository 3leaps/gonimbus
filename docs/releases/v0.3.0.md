# v0.3.0 Release Notes

## Summary

v0.3.0 changes the default index-store build from CGO/libsql to pure-Go
`modernc.org/sqlite`. Local SQLite-backed index workflows continue to work by
default. Operators who use remote libsql/Turso databases must build with
`-tags gonimbus_libsql`.

## Highlights

- Default `go build` now selects the pure-Go SQLite driver regardless of
  `CGO_ENABLED`.
- The libsql driver remains available behind the explicit
  `gonimbus_libsql` build tag.
- Library-consumer dependency boundaries are CI-checked with an embedding
  fixture that keeps storage-free packages out of the index-store compile
  graph.
- The SQLite substrate dependencies were refreshed for this release line.
- Local index databases migrate timestamp TEXT columns to fixed-width UTC
  values so SQLite ordering and range filters remain chronological.
- `pkg/opcheckpoint` adds the Experimental sensitive-local operation checkpoint
  substrate used by the forthcoming long-running resume flow.

## Library API

No Stable library package changes are introduced by this driver flip.
`pkg/indexstore`, `pkg/opcheckpoint`, and `pkg/reflowstate` remain
Experimental storageful packages.

## Verification

- `go test ./...`
- `go test ./... -tags gonimbus_libsql`
- `GOOS=linux GOARCH=arm64 go build ./...`
